Git Branching in a nutshell

December 4, 2020 / by Vu Bui Khanh Linh

You can simply understand branching like this: (image below)

A master branch/main branch contains the official/concluded file that your team/you have agreed on.

A feature branch is the branch you work on locally on your pc.

A hot fix branch is created to fix the bug you detect on the master branch.


Some basic branch commands:

Create a new branch:

git checkout -b branchname


Switch branch

git checkout branchname


List all branches you are having >

git branch

Delete branch

git branch -d branchname


Push all branches to the repo >

git push –-all origin

Show the locations of where you fetch/push data:

git remote -v


Push to a new-created branch to the repo:>

git push -u origin branch_name

-u : upstream/default branch you get the data from

origin: repo’s URL


Delete a branch in the repo locally (from your pc)

git push origin: branchname-you-want-to-delete


Merge your branch to the main branch:

git merge
git diff [fist_branch] [second_branch]


Show the file differences which aren’t yet staged (committed)

git diff
git diff [fist_branch] [second_branch]
